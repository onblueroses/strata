# Copywriting

Conversion-focused copy. Kill the AI marketing voice.

## Core Rules

### 1. Benefits over features

| Feature (bad) | Benefit (good) |
|---------------|----------------|
| "AI-powered analysis" | "Find problems in your code before your users do" |
| "Real-time collaboration" | "Your whole team edits the same document - no merge conflicts" |
| "99.9% uptime" | "Your site stays online when it matters most" |

Ask: "So what?" after every feature statement. The answer is the benefit.

### 2. Specific over vague

| Vague (bad) | Specific (good) |
|-------------|-----------------|
| "Save time" | "Set up in 5 minutes, not 5 hours" |
| "Powerful features" | "Run 20 lint rules across every file in your project" |
| "Trusted by thousands" | "Used by 2,400 teams shipping production code" |

Numbers, timeframes, and concrete outcomes beat adjectives every time.

### 3. Active voice, short sentences

| Passive/long (bad) | Active/short (good) |
|--------------------|---------------------|
| "Your project structure is validated by our tool" | "We validate your project structure" |
| "It has been designed to be used by..." | "Built for developers who..." |
| "The system provides the ability to..." | "You can..." |

**Target**: 15-20 words per sentence average. Break up anything over 25 words.

### 4. One idea per section

Each section of a landing page should communicate ONE clear idea:
- Hero: what it is + who it's for
- Problem: the pain point
- Solution: how you solve it
- Proof: why they should believe you
- CTA: what to do next

Don't mix proof into the hero. Don't explain the solution in the problem section.

## Banned Words and Phrases

These are AI copy tells. Replace every one:

| Banned | Why | Use instead |
|--------|-----|-------------|
| "Revolutionize" | Meaningless hyperbole | Describe the actual change |
| "Seamless" | Used on everything, means nothing | Describe what's actually smooth |
| "Cutting-edge" | Every AI tool claims this | Be specific about what's new |
| "Leverage" | Corporate jargon | "Use" |
| "Empower" | Vague, patronizing | Describe the capability gained |
| "Robust" | AI's favorite adjective | Describe what makes it robust |
| "Streamline" | Vague process claim | "Reduce X from 5 steps to 1" |
| "Unlock" | Metaphor overload | "Get access to" or just describe the feature |
| "Game-changing" | Hyperbole | Show the change with data |
| "Best-in-class" | Unverifiable claim | Compare to a specific alternative |
| "Comprehensive" | Lazy substitute for listing what's included | List the actual things |
| "Ecosystem" | Used when "tools" or "integrations" would be clearer | Name the specific integrations |

## Page-Type Patterns

### Landing page structure

1. **Hero**: Clear headline (what + who) + subheadline (key benefit) + CTA
2. **Problem**: 2-3 pain points the target audience recognizes
3. **Solution**: How your product addresses each pain point
4. **Social proof**: Testimonials, logos, numbers (real ones)
5. **Features**: 3-4 key features with benefits (not feature lists of 12+)
6. **CTA**: Repeat the call to action. One clear action.

### Headline formulas

- **[Do X] without [pain point Y]**: "Deploy your app without touching a terminal"
- **[X] for [audience]**: "Code review for teams that ship daily"
- **[Specific benefit] in [timeframe]**: "Clean project structure in 60 seconds"
- **Stop [pain]. Start [gain]**: "Stop debugging config. Start building features."

## Tone Calibration

| Audience | Tone | Example |
|----------|------|---------|
| Developers | Direct, technical, no fluff | "Run `strata init` and you're done." |
| Designers | Visual, outcome-focused | "Every page looks intentional, not templated." |
| Business/managers | ROI, risk reduction | "Ship with confidence. Catch issues before your users do." |
| Enterprise | Trust, compliance, support | "SOC 2 compliant. 24/7 support. On-premise option." |

## Anti-Examples

| Bad | Why | Better |
|-----|-----|--------|
| "Our revolutionary AI-powered platform leverages cutting-edge technology" | 4 banned words in one sentence | "[Product] finds bugs in your code using static analysis" |
| Feature list of 15 items | Overwhelming, nothing stands out | Pick 3-4 strongest, link to full list |
| "Sign up for our newsletter" as primary CTA | Low-value, low-conversion ask | "Try it free" or "See it in action" |
| Generic testimonial: "Great product!" | No specifics, could be about anything | "Caught 12 broken links we missed in manual review - Name, Role" |

## Quality Self-Check

1. Zero banned words/phrases?
2. Every feature statement has a corresponding benefit?
3. Specific numbers/timeframes instead of vague adjectives?
4. One clear CTA per page section?
5. Sentences average 15-20 words (no walls of text)?
6. Read the hero headline aloud - does it make sense in 3 seconds?
